## 缘起

一直想拥有一个属于自己的博客空间，用来记录技术学习心得和生活思考。经过一番调研，我选择了 GitHub Pages + Jekyll 的方案，因为它完全免费、稳定可靠，而且与我的开发者工作流完美契合。

## 第一步：创建核心仓库

在 GitHub 上创建了一个特殊命名的仓库，格式必须是 `用户名.github.io`。这是我的博客能够通过专属域名访问的关键。

创建时记得勾选 "Add a README file"，这样仓库初始化后就能立即看到效果。

## 第二步：开启 Pages 功能

进入仓库的 Settings → Pages 页面，在 Build and deployment 部分选择从 main 分支的根目录部署。保存后等待几分钟，访问我的 `用户名.github.io` 域名，果然看到了 README 文件的内容！

## 第三步：选择主题美化

GitHub 提供了多种内置主题，但是我选择自己编写前端文件（其实是ai）以求更高的自由度。

## 第四步：发布第一篇文章

创建了 `_posts` 文件夹，这是我的所有博客文章的存放地。文章命名必须遵循 `年-月-日-标题.md` 的格式。

我在第一篇文章的顶部添加了 Front Matter，包括布局类型、标题、日期、分类和标签。然后用 Markdown 语法写下了开篇词，提交后很快就在博客首页看到了这篇文章。

## 第五步：个性化定制

### 添加关于页面
创建了 about.md 页面，介绍了自己和博客的定位，让访客能更好地了解这个空间。

### 配置基本信息
编辑 _config.yml 文件，设置了博客标题、作者名、描述信息等，让博客真正成为我的个人品牌。

## 工作流建立

现在我的写作流程已经规范化：
1. 在 _posts 文件夹创建新的 Markdown 文件
2. 编写内容并添加适当的分类标签
3. 通过 Git 提交更改
4. 推送到 GitHub 仓库
5. 等待自动构建完成，博文就发布了
